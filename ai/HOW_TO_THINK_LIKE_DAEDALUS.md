# How To Think Like Daedalus

Read this before writing code.

This document is not the Constitution.

It is not the Manifesto.

It is not Architecture.

It exists to give you the correct mental model before you begin.

If you think about Daedalus incorrectly, you will build the wrong software.

## First Principle

Daedalus is not trying to build better software.

Daedalus is trying to help humans understand reality.

Everything else is a consequence.

## The Product

The Digital Twin is the Product.

Not the report.

Not the conversation.

Not the PDF.

Not the animation.

Not the AR experience.

Not the recommendation.

Everything else is a projection of the Twin.

## The Twin Is A Place

Do not think of the Twin as a page.

Do not think of the Twin as a report.

Do not think of the Twin as a dashboard.

Think of it as a place.

The user goes to their home.

They explore it.

They change layers.

They ask questions.

They compare futures.

The Twin always remains the centre of the experience.

## The House Is The Interface

The user should interact with:

- rooms
- components
- pipework
- services
- evidence
- uncertainty

The user should not spend their time interacting with:

- cards
- dashboards
- inspector panels
- forms
- reports

The UI should disappear behind the Twin.

## Navigation

Think Google Earth.

Not a website.

Users should be able to:

- zoom
- rotate
- tilt
- isolate floors
- hide layers
- reveal systems
- reveal evidence
- reveal uncertainty

The camera moves.

The Twin remains.

## Capture

Capture is CSI.

Capture observes.

Capture records.

Capture preserves.

Capture reconstructs reality.

Capture does not solve.

Capture does not explain.

Capture does not recommend.

Capture uses both additive and deductive reasoning.

Every observation should either:

- improve the Twin
- eliminate incorrect interpretations
- identify the next most valuable observation

Capture maximises information gain.

Not evidence quantity.

## Main

Main explains reality.

Main does not create truth.

Main works only from the Twin.

Every explanation must remain traceable to evidence.

If the explanation cannot be traced back to reality, it should not exist.

## Reports

Reports are frozen projections.

Reports never become truth.

They are simply another view of the Twin.

The same is true for:

- conversations
- accessibility narration
- AR
- customer views
- engineer views
- installer views
- PDFs

One Truth.

Many Views.

## Scenarios

Daedalus does not compare products.

Daedalus compares futures.

The question is never:

"What boiler should I buy?"

The question is:

"What changes if reality changes?"

## Goals

Customers rarely want products.

Customers want outcomes.

Understand goals first.

Model reality second.

Explain consequences third.

Humans decide.

## Physics

Physics lives in relationships.

Components define capability.

Relationships define behaviour.

The interesting part of a system is often not the object itself.

It is the interaction.

## Evidence

Evidence is attached to reality.

Photos.

Voice notes.

Measurements.

Documents.

Human observations.

Evidence never floats independently from the Twin.

## Review

Review is not metadata review.

Review is reality review.

The surveyor should finish by thinking:

"Yes. That now looks like the property I have just surveyed."

Not:

"I have confirmed all the fields."

## Language

Avoid:

- Best
- Better
- Recommendation
- Suitable
- Upgrade
- Instant
- Optimised

Prefer:

- Consequence
- Change
- Constraint
- Challenge
- Behaviour
- Service
- Goal
- Transformation
- Evidence
- Confidence
- Uncertainty

## User Experience

The user should think about the home.

Not the software.

If the user spends more time looking at UI than the Twin, the design has
failed.

## The Final Test

Before implementing any feature ask:

- Does this improve the Digital Twin?
- Does this improve understanding?
- Is it objectively grounded?
- Could another projection present the same truth?
- Does it help the user understand their home better?

If the answer is "no", stop and think again.

## Remember

Daedalus is not software about homes.

Daedalus is a place where people come to understand their home.

Build that place.
